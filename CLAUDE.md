# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Overview

Human Activity Recognition (HAR) project for STM32F767IGT6 (Cortex-M7, LQFP176). A 1D-CNN TensorFlow Lite model classifies 6-axis IMU data (ax,ay,az,gx,gy,gz) into 7 activity classes: walking, running, sitting, standing, upstairs, downstairs, unknown.

## Build & Development

### STM32CubeIDE (primary)

- Open the project: `File → Import → Existing Projects` and select the project root directory. The project name inside CubeIDE is `finalwork`.
- **Build:** `Ctrl+B` or `Project → Build Project`
- **Clean build:** `Project → Clean...` then Build
- **Flash to board:** `Run → Debug` or `Run → Run` (requires ST-Link connected)
- The `.ioc` file (`finalwork.ioc`) is the CubeMX config. To regenerate code: open `.ioc` in CubeMX/CubeIDE → verify peripherals → `Generate Code`. **After generation, `Src/app_x-cube-ai.c` and `Inc/constants_ai.h` get reset to empty templates — they MUST be restored.**

### Python training

```powershell
py -3 -m pip install -r training/requirements.txt
py -3 training/train_har.py --data-dir data/raw --output-dir models/har
```

Outputs: `har_model.keras`, `har_model.tflite`, `labels.json`, `normalization.json`, `training_summary.json`.

### Model deployment to MCU

When CubeMX X-CUBE-AI fails to generate network C files, use the CLI directly:

```bash
# Paths
STEDGEAI="C:/Users/Sangonomiya KJ/STM32Cube/Repository/Packs/STMicroelectronics/X-CUBE-AI/10.2.0/Utilities/windows/stedgeai.exe"
MODEL="E:/har_model/har_model.tflite"   # short path, no Chinese characters
OUTPUT="E:/Desktop/HAR/har_project/.ai/network_output"

# Generate
"$STEDGEAI" generate --target stm32f7 --name network_3 -m "$MODEL" --compression none --output "$OUTPUT"

# Copy to project
cp "$OUTPUT"/network_3*.c Src/
cp "$OUTPUT"/network_3*.h Inc/
```

- **Use `.tflite`, not `.keras`** — ST Edge AI does not support Keras 3 format.
- **Avoid paths with Chinese characters or >260 chars** — ST Edge AI tools fail on Windows long-path limits.

## Architecture

### Model inference pipeline

```
Sensor data (UART/SD/USB/ETH)
  → HAR_Process() [har_app.c] polls input hooks, parses "ax,ay,az,gx,gy,gz" lines
  → Sliding window: 128 samples × 6 axes, stride 64 (Inc/har_app.h)
  → HAR_RunNetwork() [app_x-cube-ai.c] calls ai_network_3_run()
  → ai_network_3_run() [Src/network_3.c] executes the TFLite model via X-CUBE-AI runtime
  → Output: "HAR,source=UART,label=walking,confidence=0.680\r\n"
```

### Key files

| File | Role |
|------|------|
| `Src/main.c` | Entry point, calls `MX_X_CUBE_AI_Init()` then loops `MX_X_CUBE_AI_Process()` |
| `Src/app_x-cube-ai.c` | **Network lifecycle**: creates network_3 handle in init, overrides `HAR_RunNetwork()` with real inference. **Must be restored after each CubeMX code generation.** |
| `Src/har_app.c` | **Application logic**: sliding window buffer, input hook polling, result formatting. I/O hooks are `HAR_WEAK` — override them in a board file after drivers are ready. |
| `Inc/har_app.h` | Window/class constants. `HAR_WINDOW_SAMPLES=128`, `HAR_CLASS_COUNT=7`, `HAR_AXIS_COUNT=6`. |
| `Src/network_3.c` | Generated by ST Edge AI. Contains the compiled TFLite model as C arrays and layer functions. Do not hand-edit. |
| `Inc/network_3.h` | Generated API: `ai_network_3_create_and_init()`, `ai_network_3_run()`, buffer size defines. |
| `Src/syscalls.c` | Minimal newlib bare-metal stubs (`_write`, `_read`, `_close`, etc.) |
| `training/train_har.py` | 1D-CNN (Conv1D×2 + BatchNorm + MaxPool + GlobalAvgPool + Dense+Softmax). 1016 params, float32. |

### Weak function override pattern

`har_app.c` defines I/O hooks as `HAR_WEAK` (resolves to `__attribute__((weak))` on GCC). Board-specific implementations in separate files will automatically override them at link time. `HAR_RunNetwork()` is already overridden by `app_x-cube-ai.c` (non-weak definition).

### AI runtime library

- **Location:** `Middlewares/ST/AI/` — copied manually from X-CUBE-AI pack
  - `Inc/` — 93 runtime headers (`ai_platform.h`, `ai_math_helpers.h`, etc.)
  - `Lib/` — `libNetworkRuntime1020_CM7_GCC.a` (must have `lib` prefix for GCC linker)
- **CubeIDE setup:** Include path `Middlewares/ST/AI/Inc`, library search path `Middlewares/ST/AI/Lib`, library `NetworkRuntime1020_CM7_GCC`

## Critical Gotchas

### After CubeMX code generation

CubeMX regeneration **resets** these files. They must be restored from a backup or re-created:

1. **`Src/app_x-cube-ai.c`** — network init (`ai_network_3_create_and_init`), `HAR_RunNetwork()` override
2. **`Inc/constants_ai.h`** — `AI_NETWORK_3_DATA_ACTIVATIONS_SIZE 7296`
3. **`Middlewares/ST/AI/`** — the entire directory gets wiped

### Build exclusions

These CMSIS source folders must be **excluded from build** in CubeIDE (right-click → Resource Configurations → Exclude from Build):

- `Drivers/CMSIS/RTOS2` — not using RTOS
- `Drivers/CMSIS/NN` — not using CMSIS-NN
- `Drivers/CMSIS/DSP` — not using CMSIS-DSP
- `Drivers/CMSIS/Core_A` — for Cortex-A, not Cortex-M7
- `Drivers/CMSIS/Core/Template` — ARMv8-M template, not STM32F7

### Compiler

- **Keil MDK-ARM:** ARM Compiler 5 (`armcc`) is incompatible with the X-CUBE-AI runtime library (compiled for ARM Compiler 6 `armclang`). Also, MDK-Lite has a 32KB code size limit that this project exceeds (~36KB).
- **CubeIDE:** GCC 14.3.rel1 (`arm-none-eabi-gcc`). No code size limit. This is the supported toolchain.

### Startup file

CubeIDE requires the **GCC** startup file, not the ARMCC/Keil one. The correct file is at:
```
Src/startup_stm32f767xx.s  (sourced from STM32Cube_FW_F7/Templates/gcc/)
```
The `MDK-ARM/startup_stm32f767xx.s` is ARMCC syntax and will not link correctly with GCC.

### Model inputs

- Input tensor: `[1, 128, 6]` float32 (128 time steps × 6 sensor axes)
- Input flattening order: `[sample0_ax, sample0_ay, ..., sample0_gz, sample1_ax, ...]`
- Output tensor: `[1, 7]` float32 (softmax scores)
- Requires the same per-axis normalization (mean/std) computed during training (saved in `normalization.json`). Currently not applied at inference — add before `HAR_RunNetwork()` call.
